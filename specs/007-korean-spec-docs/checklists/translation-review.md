# Spec Kit 번역 검토 기록

## 목적

번역 배치마다 G0~G5를 같은 기준으로 검증하고, 기계 검사와 사람의 의미 검토 결과를 재현 가능하게 기록한다.

## 사용 규칙

1. 배치를 시작하기 전에 G0 입력과 선행 배치 상태를 확인한다.
2. 번역이 끝나면 G1부터 G5까지 순서대로 검증한다.
3. 하나라도 실패하면 다음 배치로 넘어가지 않고 원인 게이트로 돌아간다.
4. `pending` 상태는 아직 실행하지 않은 배치를 뜻하며 실패와 구분한다.
5. 원문 설계 문제는 번역에서 수정하지 않고 `translation-inventory.md`의 발견 사항에 기록한다.

## 보호 요소 분류

| 분류 | 보호 대상 | 기본 판정 |
|------|-----------|-----------|
| 정확 일치 | 요구사항·작업 ID, 경로, 명령어, URL, 데이터 필드명, 정확한 면책문 | 승인 없는 문자 변경은 실패 |
| 구조 보존 | 제목 계층, 순서 있는 목록, 표의 열·행 대응, 링크 목적지, 코드 펜스 | 누락·재배열·깨진 참조는 실패 |
| 의미 보존 | 의무 강도, 부정문, 숫자·횟수·기간, 우선순위, 실패·대체 조건, 범위 경계 | 누락·완화·추가는 실패 |

| 게이트 | 확인 항목 | 합격 조건 | 실패 시 처리 |
|--------|-----------|-----------|--------------|
| G0 기준선 | 대상 경로, 산출물 유형, 원문, 용어, 보호 요소, 직접 의존 문서, 선행 배치 | 모든 입력이 존재하고 선행 배치가 승인됨 | 배치를 시작하지 않음 |
| G1 구조 | 제목, 목록, 표, ID, 숫자, 링크, 코드 펜스, 명령어 | 승인되지 않은 구조·보호 요소 변경 0건 | 번역 단계로 반환 |

| G5 완료 | 공백, 자리표시자, 상대 링크, 결과 기록, diff | 미완성·깨진 참조·기록 누락 0건 | 원인 게이트로 반환 |

## 검증 방법

| 대상 | 방법 | 결과 근거 |
|------|------|-----------|
| 문서 목록 | `find specs -type f -name '*.md' -print | sort`와 inventory 비교 | 경로 개수와 차집합 |
| ID | Git 원문과 `(FR|SER|SC)-[0-9]{3}|T[0-9]{3}` 집합 비교 | `diff -u` 출력 |
| 숫자 | Git 원문과 숫자·백분율 후보 목록 비교 | 차이의 문맥 검토 |
| 코드 펜스 | 원문과 코드 펜스 구분자의 쌍 및 내용 비교 | 펜스 수와 코드 diff |
| 링크 | 표시 문구와 목적지를 분리해 상대 경로 존재 여부 확인 | 누락 경로 목록 |
| 언어 | `[A-Za-z]{4,}` 후보를 용어집·보호 목록과 대조 | 미번역 설명 목록 |
| 의미 | `git diff --word-diff` 문단 검토 | 검토 메모 |
| 형식 | `git diff --check` | 종료 코드와 출력 |

## 정확 일치 검증 규칙

| 보호 대상 | 추출·비교 기준 | 실패 조건 |
|-----------|----------------|-----------|
| 요구사항 ID | `(FR|SER|SC)-[0-9]{3}` 집합을 원문과 비교 | ID 누락·추가·번호 변경 |
| 작업 ID | `T[0-9]{3}` 집합을 원문과 비교 | ID 누락·추가·번호 변경 |
| 우선순위·스토리 표기 | `P[0-9]`, `[P]`, `[US[0-9]+]` 후보를 비교 | 의미 있는 표기 누락·변경 |
| 숫자·비율·기간 | 숫자, 소수, 백분율과 인접 단위를 원문 문맥과 비교 | 값·단위·최대/최소 의미 변경 |
| 링크 | Markdown 링크의 괄호 안 목적지와 대상 파일 존재 여부 비교 | 목적지 변경, 대상 누락, 깨진 앵커 |
| 파일 경로 | 인라인 코드와 코드 블록의 `specs/`, `src/`, `tests/` 등 경로 비교 | 승인 없는 경로 변경 |
| 명령어 | 셸 코드 블록과 인라인 명령의 실행 토큰 비교 | 실행 의미·옵션·인자 변경 |
| 데이터 필드명 | snake_case, 타입명, 테이블·열·상태 값 후보 비교 | 계약 식별자 번역·변경 |
| 코드 펜스 | 펜스의 개수·언어 태그·짝과 실행/계약 코드 내용 비교 | 펜스 불균형, 태그·코드 의미 변경 |
| 면책문 | 아래 정확 문장을 문자 단위로 비교 | 글자·공백·문장부호 변경 또는 누락 |

정확한 면책문:

```text
본 보고서는 교육 및 정보 제공 목적의 AI 리서치 결과이며, 특정 종목의 매수·매도·보유를 권유하지 않습니다. 최종 투자 판단과 책임은 투자자 본인에게 있습니다.
```

## 구조 및 의미 검증 규칙

- 제목은 번역할 수 있지만 상대적인 `#` 계층과 섹션 순서를 보존한다.
- 순서 있는 목록의 번호가 실행 순서나 우선순위를 나타내면 순서를 보존한다.
- 표 제목과 셀은 번역할 수 있지만 열의 수, 행 대응, 식별자와 값의 연결을 보존한다.
- `MUST`, `MUST NOT`, `SHOULD`, `MAY`는 `glossary.md`의 의무 강도 규칙에 따라 번역한다.
- 부정문, 제외 범위와 금지 조건은 삭제하거나 권고 수준으로 완화하지 않는다.
- 인수 시나리오의 전제·행동·결과와 성공 기준의 측정 조건을 보존한다.
- 원문에 없던 설계 결정이나 구현 범위를 번역문에 추가하지 않는다.

## Constitution 중요 의미 검증 규칙

G4에서는 대상 문서에 관련 내용이 있을 때 다음을 각각 확인한다.

| 영역 | 반드시 보존할 의미 | 실패 조건 |
|------|--------------------|-----------|
| 금융 안전 | 직접적인 매수·매도·보유 권유, 거래 지시, 실거래 실행 금지 | 금지가 누락·완화되거나 실행 범위가 추가됨 |
| 보장 표현 | 수익, 목표가, 원금 보장과 손실 없음 주장 금지 | 보장 금지가 누락·완화됨 |
| 시나리오 | 관망, 분할 접근, 리스크 회피 시나리오 표현 | 직접 투자 권유로 변경됨 |
| 근거 | 중요 수치·사실의 `EvidenceItem` 연결과 근거 요약 | 근거 의무 또는 필드 의미가 누락됨 |
| 입력 검증 | ticker 검증 실패 시 사용자 친화적 오류로 중단 | 실패가 정상 진행으로 변경됨 |
| 시장 데이터 | 실패 시 1회 재시도 후 성능 저하 보고 | 횟수·종료·누락 공개 조건 변경 |
| 뉴스 데이터 | 실패 시 모의 뉴스 대체 또는 뉴스 없음 경고 | 실패가 근거 없는 뉴스 생성으로 변경됨 |
| 평가·재작성 | 평가 실패 시 재작성 후 재평가, 최대 2회 재작성 | 경로·최대 횟수·종료 조건 변경 |
| 데이터 한계 | 누락·오래된 데이터와 한계를 명시 | 정상 데이터로 오해하게 표현됨 |
| 관측성 | run_id, ticker, 노드, 시작·종료, 성공·실패, 오류, 평가 점수 로그 | 필수 필드가 누락됨 |
| 지표 | total_runs, successful_runs, failed_runs, average_evaluation_score | 필수 지표명·의미가 변경됨 |
| 결정적 검증 | 외부 실시간 API 없는 재현 가능한 테스트 | live API 의존이 추가됨 |

G4와 무관한 문서는 `not_applicable`로 기록할 수 있지만, 판단 근거를 남겨야 한다.
