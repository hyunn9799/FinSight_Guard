# 계약: Spec Kit 문서 번역 및 검증

## 목적

이 계약은 외부 API 계약이 아니라 번역 작업자와 검토자 사이의 내부 품질 계약이다. 각 번역 배치가 받아야 할 입력, 만들어야 할 출력, 보존해야 할 요소와 합격 조건을 정의한다.

## 배치 입력 계약

번역을 시작하려면 다음 입력이 준비되어야 한다.

| 입력 | 필수 여부 | 조건 |
|------|-----------|------|
| 대상 문서 경로 | 필수 | `specs/` 아래의 실제 Markdown 파일 |
| 원문 기준선 | 필수 | Git 이력 또는 승인된 직전 버전에서 조회 가능 |
| 공통 용어 항목 | 필수 | 대상 문서에 등장하는 핵심 용어가 포함됨 |
| 보호 요소 목록 | 필수 | ID, 숫자, 링크, 코드 펜스, 명령어와 constitution 중요 문구 포함 |
| 직접 의존 문서 | 필요한 경우 | 의미 또는 링크 관계가 있는 최소 문서만 포함 |
| 선행 배치 상태 | 필수 | 모든 선행 배치가 `approved` |

입력이 하나라도 충족되지 않으면 번역 배치는 `blocked`로 처리한다.

## 번역 출력 계약

승인 후보 문서는 다음 조건을 충족해야 한다.

1. 설명 문장, 제목, 근거, 시나리오와 검토 메모가 자연스러운 한국어다.
2. 영문 원문을 주석이나 병렬 본문으로 중복하지 않는다.
3. 기술 식별자, 필드명, 경로, 명령어, URL과 표준 약어는 필요할 때 원형을 유지한다.
4. 원문의 섹션 순서, 우선순위, 인수 조건, 범위와 의무 강도를 보존한다.
5. 원문 문제를 발견하면 번역과 분리된 발견 사항으로 기록한다.
6. 변경된 각 문서에는 해당 배치의 검증 결과를 연결할 수 있다.

## 보호 요소 계약

### 정확 일치

다음 요소는 승인 없이 변경할 수 없다.

- `FR-*`, `SER-*`, `SC-*`, `T*` 식별자
- 코드 식별자와 데이터 필드명
- 파일 경로, 명령어와 URL
- 링크 목적지
- 실행 또는 데이터 계약을 표현하는 코드 블록
- 다음 면책문

```text
본 보고서는 교육 및 정보 제공 목적의 AI 리서치 결과이며, 특정 종목의 매수·매도·보유를 권유하지 않습니다. 최종 투자 판단과 책임은 투자자 본인에게 있습니다.
```

### 구조 보존

- 제목의 상대적인 계층
- 순서가 의미를 갖는 목록과 단계
- 표의 열 의미와 행 대응
- 코드 펜스의 시작·종료 쌍
- 문서 간 상대 참조

### 의미 보존

- MUST, MUST NOT, SHOULD 등에 해당하는 의무 강도
- 부정 표현과 금지 범위
- 숫자, 백분율, 횟수, 기간과 우선순위
- 실패 조건, 재시도 횟수, 대체 처리와 종료 조건
- 포함 범위, 제외 범위와 소유권 경계
- 인수 시나리오와 성공 기준

## 게이트 계약

| 게이트 | 검증 대상 | 합격 조건 | 실패 시 처리 |
|--------|-----------|-----------|--------------|
| G0 기준선 | 대상·용어·보호 요소·의존 문서 | 모든 입력이 기록되고 선행 배치가 승인됨 | 배치를 시작하지 않음 |
| G1 구조 | ID, 숫자, 링크, 제목, 표, 코드 펜스, 명령어 | 승인되지 않은 삭제·변경 0건 | 번역 단계로 반환 |
| G2 언어 | 설명 문장과 허용된 원문 용어 | 미번역 설명 문장 0건, 허용된 원문만 남음 | 번역 단계로 반환 |
| G3 의미 | 의무, 부정, 범위, 조건, 시나리오 | 의미 누락·완화·추가 0건 | 의미 수정 후 재검토 |
| G4 constitution | 안전, 근거, 실패 처리, 관측성 | 중요 원칙 누락·완화 0건, 정확 문구 일치 | 전체 배치 실패 |
| G5 완료 | 형식, 참조, 체크리스트, diff | 미완성 표시·깨진 참조·공백 오류 0건, 모든 체크 완료 | 원인 게이트로 반환 |

## 영어 문자열 판정 계약

영어 문자열 검색은 번역 후보를 찾는 보조 검사다. 다음은 영어가 남아 있어도 자동 실패가 아니다.

- 코드와 데이터 식별자
- 경로, 명령어, URL
- 제품명, 라이브러리명, 표준 약어
- 용어집에서 원문 유지 또는 최초 병기를 허용한 용어
- 코드 블록과 계약 예시에서 정확성이 필요한 값

검색 결과는 문장 단위로 검토하여 설명 문장인지 보호 요소인지 분류한다. 단순 영문 문자 수나 비율은 합격 기준으로 사용하지 않는다.

## 실패와 예외 계약

- 보호 요소가 승인 없이 바뀌면 해당 배치는 실패한다.
- 원문과 constitution이 충돌하면 번역으로 해결하지 않고 배치를 차단한 뒤 별도 검토한다.
- 상대 링크 대상이 저장소에 없으면 누락 사유가 확인될 때까지 승인하지 않는다.
- 기술 용어 번역이 여러 방식으로 가능한 경우 용어집 결정을 먼저 갱신하고 전체 배치에 일관되게 적용한다.
- 예외 승인은 대상, 이유, 검토한 대안과 영향 범위를 기록해야 한다.

## 배치 완료 출력

승인된 배치는 다음 결과를 남긴다.

- 한국어화된 문서
- 갱신된 번역 상태와 용어 항목
- G0~G5 결과
- 발견된 원문 문제와 후속 조치
- 다음 배치가 재사용할 보호 요소와 용어 결정
